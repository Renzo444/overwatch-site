VS Code Remote-SSH RCE Lets Attackers Pivot From Developer Machines to Cloud Servers

By Overwatch Observatory — Automated Detection Engineering

Threat Overview

Cyber Security News recently reported on a critical vulnerability in VS Code's Remote-SSH extension, detailing a Time-of-Check to Time-of-Use (TOCTOU) race condition. This attack vector allows an adversary, already present on a developer's machine, to pivot to remote cloud servers by manipulating bootstrap scripts. This represents a significant risk for lateral movement (T1021.004) and execution flow hijacking (T1574) within trusted development workflows.

Our analysis focused on detecting the behavioral indicators of this attack. We developed a KQL query that leverages `DeviceFileEvents` to identify suspicious activity. Specifically, the query looks for instances where a shell script (`.sh` file) is created in common temporary directories like `/tmp/` or `/var/tmp/`, or in VS Code-specific paths such as `/.vscode-server/` or `/tmp/vscode-ssh-remote-`, and then rapidly modified by a *different* process within a 10-second window. The query first identifies `FileCreated` events for these scripts and then joins them with `FileModified` events for the same file on the same device. The critical part of our detection logic is ensuring that the `InitiatingProcessId` for the creation event is distinct from the `InitiatingProcessId` for the modification event. This helps us pinpoint scenarios where a legitimate process creates a temporary script, and an attacker-controlled process quickly alters it before execution on the remote server.

Defenders should deploy this KQL query in their Microsoft Defender for Endpoint environments to monitor for these specific TOCTOU patterns. We recommend reviewing any alerts generated by this rule, paying close attention to the `CreatingProcessName` and `ModifyingProcessName` fields to understand the processes involved. Tuning the `timeWindow` variable might be necessary based on environmental specifics to reduce false positives while maintaining detection efficacy.

Detection Rules

TA0005 / T1574

Monitors file system activity for behavioral patterns associated with T1574 (TA0005). Deploy this query in Microsoft Sentinel to surface suspicious activity in your environment.

KQL 
let timeWindow = 10s;
let tempFolders = dynamic(["/tmp/", "/var/tmp/"]);

DeviceFileEvents
| where TimeGenerated > ago(1d)
| where ActionType == "FileCreated" and FileName endswith ".sh"
| where FolderPath has_any (tempFolders) or FolderPath contains "/.vscode-server/" or FolderPath contains "/tmp/vscode-ssh-remote-"
| project CreatedTime=TimeGenerated, FilePath=strcat(FolderPath, FileName), CreatingProcessId=InitiatingProcessId, CreatingProcessName=InitiatingProcessFileName, DeviceId, ReportId_Created=ReportId

| join kind=inner (
    DeviceFileEvents
    | where TimeGenerated > ago(1d)
    | where ActionType == "FileModified" and FileName endswith ".sh"
    | where FolderPath has_any (tempFolders) or FolderPath contains "/.vscode-server/" or FolderPath contains "/tmp/vscode-ssh-remote-"
    | project ModifiedTime=TimeGenerated, FilePath=strcat(FolderPath, FileName), ModifyingProcessId=InitiatingProcessId, ModifyingProcessName=InitiatingProcessFileName, DeviceId, ReportId_Modified=ReportId
) on DeviceId, FilePath

| where ModifiedTime between (CreatedTime .. (CreatedTime + timeWindow))

| where CreatingProcessId != ModifyingProcessId
| project
    TimeGenerated = ModifiedTime,
    Timestamp_Created = CreatedTime,
    Timestamp_Modified = ModifiedTime,
    DeviceId,
    FilePath,
    CreatingProcessName,
    CreatingProcessId,
    ModifyingProcessName,
    ModifyingProcessId,
    ReportId_Created,
    ReportId_Modified
